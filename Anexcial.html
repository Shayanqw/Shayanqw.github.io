Anexcial is a full-stack Django web application for an invite-only customer loyalty platform designed for neighbourhood businesses such as cafes and small retail stores.

The project combines a marketing website, role-based dashboards, a secure points system, invite-based onboarding, and operations tooling in one product experience.

Project vision

The vision for Anexcial is to give neighbourhood businesses a more premium and controlled loyalty experience than a typical open points app. Instead of public enrollment, the platform uses curated store onboarding, invite-only customer access, operational oversight, and trust-first rewards.

What the project does

  • Public-facing marketing pages for pricing, consultation, catalog, FAQ, privacy, and terms.
  • Invite-based member registration with store-issued access codes.
  • Store onboarding with approval and rejection workflow.
  • Member dashboard with points, rewards, transaction history, and store cards.
  • QR-based member identity for award and redemption workflows.
  • Store dashboard for points management, item catalog management, and invite creation.
  • Admin operations hub for users, stores, activity, onboarding requests, and leads.

Project and business requirements

  • Allow approved stores to manage loyalty operations independently.
  • Support invite-only membership tied to store-issued codes.
  • Maintain accurate point balances using an auditable transaction ledger.
  • Enable secure award and redemption workflows through QR-based identification.
  • Provide visibility for onboarding requests, leads, users, and store activity.
  • Reduce misuse through role-based access control, validation, and duplicate scan protection.

Planning and analysis

  • Defined user roles first so members, store staff, and admins each had a clear workflow.
  • Separated the platform into marketing, member experience, store operations, and internal admin management.
  • Prioritized server-side validation and transaction safety so the loyalty logic could not be bypassed from the client side.
  • Used screenshots to document the visual planning and implementation of the dashboard and reward experience.

Implementation highlights

  • Backend: Python and Django 5.
  • Frontend: Django templates, HTML, CSS, and vanilla JavaScript.
  • Data handling: SQLite for development with optional PostgreSQL in production.
  • Security and reliability: environment-based settings, duplicate scan protection, trusted origins, and automated testing.
  • Core models: Store, MemberProfile, StoreItem, PointsTransaction, InviteCode, StoreOnboardingRequest, and ConsultationLead.

The project reached a portfolio-ready working state with completed core flows for onboarding, invites, points management, QR identification, and admin operations. The current verification summary is 42 passing automated tests.
